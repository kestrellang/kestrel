# Type Inference Reference

Complete reference for Kestrel's constraint-based type inference system. Covers every constraint kind, every solving rule, the oracle interface, and the apply phase.

Crate: `kestrel-semantic-type-inference`

## Architecture

```
┌─────────────────────────────────────────────────────────────┐
│  TypeInferenceAnalyzer (per function/initializer/getter)    │
│                                                              │
│  1. GENERATE        constraint_generator.rs                  │
│     Walk expressions + patterns → emit Constraints           │
│                                                              │
│  2. COLLECT         context.rs (InferenceContext)            │
│     Buffer constraints, register types, track metadata       │
│                                                              │
│  3. SOLVE           solver.rs                                │
│     Fixpoint loop: try_solve each constraint until stable    │
│     Then: apply literal defaults → apply Never defaults      │
│                                                              │
│  4. EXTRACT         solution.rs                              │
│     Solution { types, values, promotions, member_kinds }     │
│                                                              │
│  5. APPLY           apply.rs                                 │
│     Rewrite Deferred* expressions → concrete expressions     │
│     Insert FromValue.from() wrappers for promotions          │
└─────────────────────────────────────────────────────────────┘
         ▲
         │ queries
         ▼
┌─────────────────────────────────────────────────────────────┐
│  TypeOracle (trait)              oracle.rs                   │
│  Implemented by ContextualOracle in kestrel-semantic-model   │
│                                                              │
│  Provides: member resolution, conformance checks,           │
│  associated type resolution, type alias expansion,           │
│  default literal types, FromValue conformance, visibility    │
└─────────────────────────────────────────────────────────────┘
```

## Constraint Kinds

10 constraint types. Each has a `try_solve` rule (see Solving Rules below).

### 1. Equals

```rust
Constraint::Equals { a: TyId, b: TyId, span: Span }
```

Two types must be structurally identical. The fundamental unification constraint.

**Generated by:** assignment, parameter/argument binding, pattern binding, return type matching, tuple element matching, array element matching.

### 2. Conforms

```rust
Constraint::Conforms { ty: TyId, protocol: ProtocolRef }
```

Type must conform to a protocol.

**Generated by:** literal expressions (`ExpressibleByIntLiteral`, etc.), generic bounds, `MethodRef` pointing at protocol methods.

### 3. Normalizes

```rust
Constraint::Normalizes { base: TyId, assoc_name: String, result: TyId, span: Span }
```

Associated type projection. Resolves `Base.AssocName` to a concrete type.

**Generated by:** array literal `Element` type, dictionary literal `Key`/`Value` types, protocol associated type access.

### 4. MemberAccess

```rust
Constraint::MemberAccess {
    receiver: TyId,
    member: String,
    is_static: bool,
    arguments: Vec<TyId>,
    labels: Vec<Option<String>>,
    is_property_access: bool,
    result: TyId,
    expr_id: ExprId,
    substitutions: Substitutions,
    explicit_type_args: Option<Vec<Ty>>,
    span: Span,
}
```

Deferred member resolution — method calls, field access, property access, subscripts, init calls.

**Generated by:** `DeferredMethodCall`, `DeferredStaticCall`, `DeferredInitCall`, `DeferredMemberAccess`, `DeferredSubscriptCall`.

### 5. ImplicitMember

```rust
Constraint::ImplicitMember {
    expr_ty: TyId,
    member_name: String,
    argument_tys: Vec<(Option<String>, TyId)>,
    expr_id: ExprId,
    span: Span,
}
```

Enum shorthand syntax: `.Case` or `.Case(args)`. Resolved when expected type becomes known.

**Generated by:** `ImplicitMemberAccess` expressions (`.some(value)`, `.none`).

### 6. EnumPatternBinding

```rust
Constraint::EnumPatternBinding {
    enum_ty: TyId,
    case_name: String,
    binding_tys: Vec<(Option<String>, TyId)>,
    span: Span,
}
```

Connects pattern variables to enum case parameter types.

**Generated by:** enum variant patterns in match/switch.

### 7. StructPatternBinding

```rust
Constraint::StructPatternBinding {
    struct_ty: TyId,
    struct_name: String,
    field_bindings: Vec<(String, TyId)>,
    has_rest: bool,
    span: Span,
}
```

Connects pattern fields to struct field types.

**Generated by:** struct destructuring patterns (`Point { x, y }`).

### 8. Promotable

```rust
Constraint::Promotable {
    from_ty: TyId,
    to_ty: TyId,
    expr_id: ExprId,
    span: Span,
}
```

Value may be implicitly promoted via `FromValue.from()`. Tries unification first; falls back to `FromValue` conformance.

**Generated by:** assignment, return, function arguments, if/else branch values.

### 9. TupleIndexAccess

```rust
Constraint::TupleIndexAccess {
    tuple: TyId,
    index: usize,
    result: TyId,
    span: Span,
}
```

Tuple element access when tuple type is deferred.

**Generated by:** tuple index expressions on deferred types.

### 10. FunctionCall

```rust
Constraint::FunctionCall {
    candidates: Vec<SymbolId>,
    arguments: Vec<TyId>,
    argument_expr_ids: Vec<ExprId>,
    labels: Vec<Option<String>>,
    explicit_type_args: Option<Vec<Ty>>,
    result: TyId,
    expr_id: ExprId,
    span: Span,
}
```

Direct function call with overload resolution.

**Generated by:** `DeferredFunctionCall` expressions.

## Constraint Generation

### Expression → Constraint Mapping

| Expression | Constraints |
|-----------|-------------|
| **Integer literal** | `Conforms(ty, ExpressibleByIntLiteral)` + register default `Int64` |
| **Float literal** | `Conforms(ty, ExpressibleByFloatLiteral)` + register default `Float64` |
| **String literal** | `Conforms(ty, ExpressibleByStringLiteral)` + register default `String` |
| **Bool literal** | `Conforms(ty, ExpressibleByBoolLiteral)` + register default `Bool` |
| **Char literal** | `Conforms(ty, ExpressibleByCharLiteral)` + register default `Char` |
| **Array literal** | `Conforms(ty, _ExpressibleByArrayLiteral)` + `Normalizes(ty, "Element", elem_ty)` + `Equals` per element |
| **Dictionary literal** | `Conforms(ty, _ExpressibleByDictionaryLiteral)` + `Normalizes` Key + Value + `Equals` per pair |
| **Tuple** | `Equals` per element |
| **Call** (resolved) | `Equals(call_ty, return_ty)` + `Promotable` per arg→param + where clause `Equals` |
| **DeferredMethodCall** | `MemberAccess(receiver, method, args, labels, result)` |
| **DeferredStaticCall** | `MemberAccess(receiver, method, args, labels, result, is_static=true)` + `Conforms` for protocols |
| **DeferredInitCall** | `MemberAccess(struct_ty, "init", args, labels, result)` |
| **DeferredMemberAccess** | `MemberAccess(receiver, member, is_property_access=true, result)` |
| **DeferredSubscriptCall** | `MemberAccess(receiver, "subscript", args, labels, result)` |
| **DeferredFunctionCall** | `FunctionCall(candidates, args, labels, result)` |
| **ImplicitMemberAccess** | `ImplicitMember(expr_ty, case_name, arg_tys)` |
| **MethodRef** | `Conforms` for protocol methods |
| **Closure** | Register metadata + `Equals` params + `Equals` return with tail |
| **Assignment** | `Promotable(value_ty, target_ty)` |
| **Return** | `Promotable(value_ty, return_ty)` |
| **If/While/Else** | `Promotable(branch_ty, expr_ty)` per branch |

### Pattern → Constraint Mapping

| Pattern | Constraints |
|---------|-------------|
| **Local** | Register type |
| **Wildcard** | Register type |
| **Literal** | `Conforms(ty, ExpressibleByXxx)` |
| **EnumVariant** | `EnumPatternBinding(enum_ty, case, bindings)` |
| **Struct** | `StructPatternBinding(struct_ty, name, fields, has_rest)` |
| **Tuple** | `Equals` per element |
| **Array** | Recursive constraints for prefix/suffix |
| **Or** | `Equals` per alternative |
| **At** | `Equals(sub_ty, at_ty)` |

## Solving Algorithm

```
solve(ctx):
  ┌─────────────────────────────────────────────────┐
  │ 1. PRE-SCAN                                     │
  │    Identify literal TyIds with default types     │
  │    Mark for deferred Promotable handling          │
  └─────────────────────────────────────────────────┘
                    │
                    ▼
  ┌─────────────────────────────────────────────────┐
  │ 2. FIXPOINT LOOP                                │
  │    repeat {                                      │
  │      progress = false                            │
  │      for constraint in constraints:              │
  │        match try_solve(constraint):              │
  │          Solved  → drop, progress = true         │
  │          Deferred → re-queue                     │
  │          Error   → record, progress = true       │
  │    } until !progress                             │
  └─────────────────────────────────────────────────┘
                    │
                    ▼
  ┌─────────────────────────────────────────────────┐
  │ 3. LITERAL DEFAULTS LOOP                        │
  │    repeat {                                      │
  │      For remaining Conforms(Infer, Expressible): │
  │        Apply default (Int64, Float64, etc.)      │
  │        Re-run fixpoint                           │
  │    } until no defaults applied                   │
  │    (handles nested: [[1,2]] → Array[Array[Int]]) │
  └─────────────────────────────────────────────────┘
                    │
                    ▼
  ┌─────────────────────────────────────────────────┐
  │ 4. NEVER DEFAULTS                               │
  │    For unresolved type params on resolved types: │
  │      Default to Never (bottom type)              │
  │    Re-run fixpoint                               │
  └─────────────────────────────────────────────────┘
                    │
                    ▼
  ┌─────────────────────────────────────────────────┐
  │ 5. COMPLETENESS CHECK                           │
  │    Error for any remaining Infer placeholders    │
  └─────────────────────────────────────────────────┘
                    │
                    ▼
               Solution
```

## Solving Rules

### Equals (Unification)

`unify(a, b)` — structural equality with substitution.

| LHS | RHS | Rule |
|-----|-----|------|
| `Infer` | `Infer` | Substitute one → the other |
| `Infer` | `T` | Substitute `Infer → T` (with occurs check) |
| `T` | `Infer` | Substitute `Infer → T` (with occurs check) |
| `Error` | `_` | Success (suppress cascading errors) |
| `_` | `Error` | Success |
| `Never` | `_` | Success (Never is bottom, coerces to anything) |
| `_` | `Never` | Success |
| `Never` | `Infer` | Success WITHOUT substituting (preserves Infer for other constraints) |
| `Infer` | `Never` | Success WITHOUT substituting |
| `Struct` | `Struct` | SymbolIds must match; recursively equate type arg substitutions |
| `Enum` | `Enum` | SymbolIds must match; recursively equate type arg substitutions |
| `Protocol` | `Protocol` | SymbolIds must match; recursively equate substitutions |
| `TypeParameter` | `TypeParameter` | SymbolIds must match |
| `Tuple` | `Tuple` | Arity must match; equate each element pairwise |
| `Pointer` | `Pointer` | Equate element types |
| `Function` | `Function` | Equate params pairwise + equate return types |
| `Closure` | `Function` | Check arity; equate param types + return type |
| `Struct` | `Protocol` | Check `conforms_to(&Struct, protocol_id)` via oracle |
| `Enum` | `Protocol` | Check `conforms_to(&Enum, protocol_id)` via oracle |
| `TypeAlias` | `_` | Expand alias via oracle, re-equate |
| `_` | `TypeAlias` | Expand alias via oracle, re-equate |
| `AssociatedType` | `_` | **Defer** (needs normalization first) |
| `_` | `AssociatedType` | **Defer** |
| `SelfType` | `SelfType` | Success |
| `SelfType` | `Struct`/`Protocol` | Success |
| `Primitive` | `Primitive` | Compare bits + kind; must match exactly |
| _other_ | _other_ | **Type mismatch error** |

### Conforms

```
check_conforms(ty, protocol):
  1. Resolve ty through substitution chain
  2. If Infer → Defer
  3. If AssociatedType with unresolved container → Defer
  4. Expand type aliases (loop)
  5. oracle.conforms_to(&ty, protocol_id)
  6. true → Solved; false → Error
```

### Normalizes

```
normalize(base, assoc_name, result):
  1. Resolve base, expand aliases
  2. If Infer → Defer
  3. oracle.resolve_associated_type(&base, assoc_name)
  4. If found → deeply resolve nested assoc types → equate with result → Solved
  5. If not found → Error
```

### MemberAccess

```
resolve_member_access(receiver, member, is_static, args, labels, result):
  1. Resolve receiver, expand aliases
  2. If Infer → Defer
  3. If Never → equate result = Never → Solved
  4. If unresolved AssociatedType → Defer

  5. Dispatch by member name:
     "init"      → oracle.resolve_init(...)
     "subscript" → oracle.resolve_subscript(...)
     property    → oracle.resolve_member(...) for field/property

  6. Regular method → oracle.resolve_member_full() or resolve_member_with_labels()

  7. On success:
     a. Visibility check via oracle.is_visible()
     b. Record ValueResolution(symbol_id, substitutions)
     c. Classify: MemberKind (Field/ComputedProperty/ProtocolProperty/Method)
     d. Merge substitutions: call-site ⊕ resolved
     e. Handle explicit type args: equate with inference vars
     f. For each arg: equate arg_ty = param_ty
     g. Extract where clause equality constraints
     h. For Self-returning methods: equate receiver = result
     → Solved
```

### ImplicitMember

```
resolve_implicit(expr_ty, member_name, arg_tys):
  1. Resolve expr_ty, expand aliases
  2. If Infer → Defer
  3. Must be Enum type (else Error)
  4. Find case by name
  5. If case has params: check arity + labels, equate each arg
  6. Record ValueResolution(case_id)
  → Solved
```

### EnumPatternBinding

```
resolve_enum_pattern(enum_ty, case_name, bindings):
  1. Resolve enum_ty, expand aliases
  2. If Infer → Defer
  3. Must be Enum
  4. Find case by name
  5. For each (label, ty_id) in bindings:
     - Find param by label/position
     - Apply type substitutions
     - equate(binding_ty, param_ty)
  → Solved
```

### StructPatternBinding

```
resolve_struct_pattern(struct_ty, name, fields, has_rest):
  1. Resolve struct_ty
  2. If Infer → Defer
  3. Must be Struct
  4. For each (field_name, ty_id):
     - Find field by name (error if missing)
     - Apply substitutions
     - equate(binding_ty, field_ty)
  5. If !has_rest: check for missing fields
  → Solved
```

### Promotable

```
resolve_promotable(from_ty, to_ty, expr_id):
  1. Resolve both, expand aliases
  2. If both Infer → unify directly
  3. If from is pure Infer:
     - If literal AND (to is Infer OR potential promotion) → Defer
     - Else → unify
  4. If to has unresolved Infer → unify
  5. Normalize type params / assoc types
  6. If types could unify → unify
  7. If can't unify:
     - oracle.check_from_value_conformance(&to, &from)
     - If found → record PromotionInfo → Solved
     - Else → type mismatch Error
```

### TupleIndexAccess

```
resolve_tuple_index(tuple, index, result):
  1. Resolve tuple, expand aliases, normalize
  2. If Infer → Defer
  3. If not Tuple → Error
  4. If index out of bounds → Error
  5. equate(result, element_ty)
  → Solved
```

### FunctionCall

```
resolve_function_call(candidates, args, labels, explicit_type_args, result):
  1. Resolve argument types
  2. oracle.resolve_function(candidates, labels, &arg_types)
  3. On success:
     - Merge substitutions + explicit type args
     - Record ValueResolution
     - equate(result, return_ty)
     - For each arg: equate(arg, param)
     - Process where clause constraints
  → Solved
```

## Type Resolution Chain

`resolve_type(ctx, id)` follows the substitution chain:

```
id → substitutions[id] → substitutions[next_id] → ... → concrete type
                                                    └── or type_registry[id]
                                                    └── or fresh Infer (last resort)
```

Special handling:
- **AssociatedType**: if container resolves, try `oracle.resolve_associated_type()`
- **Struct with type args**: recursively resolve associated types in substitutions

## TypeOracle Interface

The solver is decoupled from the semantic model via this trait:

```rust
trait TypeOracle {
    // Member resolution
    fn resolve_member(&self, receiver: &Ty, member: &str, is_static: bool) → Result<MemberResolution>
    fn resolve_member_with_labels(&self, ..., labels: &[Option<String>]) → Result<MemberResolution>
    fn resolve_member_full(&self, ..., labels, arg_types) → Result<MemberResolution>
    fn resolve_init(&self, struct_ty: &Ty, labels, arg_types) → Result<MemberResolution>
    fn resolve_subscript(&self, receiver: &Ty, labels, arg_types) → Result<MemberResolution>
    fn resolve_function(&self, candidates: &[SymbolId], labels, arg_types) → Result<MemberResolution>

    // Protocol conformance
    fn conforms_to(&self, ty: &Ty, protocol_id: SymbolId) → bool
    fn check_from_value_conformance(&self, target: &Ty, source: &Ty) → Option<(SymbolId, Subs)>

    // Type operations
    fn resolve_associated_type(&self, container: &Ty, name: &str) → Option<Ty>
    fn expand_type_alias(&self, ty: &Ty) → Ty
    fn normalize_with_constraints(&self, ty: &Ty) → Ty

    // Default literal types
    fn default_integer_type(&self, span) → Ty        // Int64
    fn default_float_type(&self, span) → Ty           // Float64
    fn default_string_type(&self, span) → Ty          // String
    fn default_boolean_type(&self, span) → Ty         // Bool
    fn default_char_type(&self, span) → Ty            // Char
    fn default_array_type(&self, elem: Ty, span) → Option<Ty>

    // Utilities
    fn symbol_name(&self, id: SymbolId) → Option<String>
    fn builtin_protocol(&self, feature: LanguageFeature) → Option<SymbolId>
    fn protocol_for_method(&self, method_id: SymbolId) → Option<SymbolId>
    fn context_symbol_id(&self) → Option<SymbolId>
    fn is_visible(&self, target: SymbolId, from: SymbolId) → bool
    fn classify_member(&self, receiver: &Ty, resolution: &MemberResolution) → MemberKind
}
```

## Solution

```rust
pub struct Solution {
    pub types: HashMap<TyId, Ty>,                    // Resolved inference variables
    pub values: HashMap<ExprId, ValueResolution>,     // Deferred member → concrete symbol
    pub promotions: HashMap<ExprId, PromotionInfo>,   // Implicit FromValue.from() wraps
    pub member_kinds: HashMap<ExprId, MemberKind>,    // Field/property classification
    pub errors: Vec<InferenceError>,                  // Type errors
}

pub struct ValueResolution {
    pub symbol_id: SymbolId,
    pub substitutions: Substitutions,
}

pub struct PromotionInfo {
    pub target_ty: Ty,
    pub from_method: SymbolId,
    pub substitutions: Substitutions,
}

pub enum MemberKind {
    Field { mutable: bool },
    ComputedProperty { has_setter: bool },
    ProtocolProperty { protocol_id: SymbolId, has_setter: bool, is_static: bool },
    Method,
}
```

## Apply Phase

Rewrites the semantic tree using the solution:

```
apply_solution(block, solution, oracle) → CodeBlock:
  Walk all expressions and statements:

  1. Replace TyKind::Infer with solution.types[id]
  2. Resolve remaining associated types via oracle

  3. Deferred → Concrete rewrites:
     DeferredMemberAccess → FieldAccess / ComputedProperty / ProtocolProperty / MethodRef
     DeferredMethodCall   → Call + MethodRef (with resolved symbol + subs)
     DeferredInitCall     → Call + TypeRef (SymbolRef for self-allocation in MIR)
     DeferredStaticCall   → Call + MethodRef
     DeferredSubscriptCall → Call + MethodRef
     DeferredFunctionCall → Call + SymbolRef
     ImplicitMemberAccess → EnumCase or Call + EnumCase

  4. Insert promotions: wrap value in FromValue.from() call
  5. Update MemberKinds on property access expressions
  6. Update MethodRef/Call with resolved ValueResolutions
```

## Key Design Patterns

### Bidirectional Type Flow
Contextual types flow backward from expected type (assignment target, return type, parameter type) to value expressions. Enables `let x: Int16 = 42` to constrain the literal to `Int16` instead of defaulting to `Int64`.

### Deferral Strategy
Constraints whose inputs aren't resolved yet are re-queued. The fixpoint loop retries until no more progress can be made. This naturally handles dependency ordering without explicit scheduling.

### Literal Defaults
Literals get `Conforms(ExpressibleByXxx)` constraints first. If still unresolved after main solving, default types are applied in a separate loop. Nested structures (e.g., `[[1, 2], [3, 4]]`) require multiple rounds — inner arrays default before outer.

### Never as Bottom Type
`Never` unifies with any type without substitution. Represents unreachable code (e.g., after `fatalError()`). Special: `Never = Infer` succeeds without substituting the Infer, allowing other constraints to resolve it to a useful type.

### Promotion Semantics
`Promotable` tries unification first. If types don't match, checks `FromValue[SourceType]` conformance on the target type. Records a `PromotionInfo` so the apply phase can insert the `.from()` wrapper call.

### Where Clause Extraction
When a method is resolved, its where clauses (e.g., `where Item == Optional[T]`) are extracted and converted to `Equals` constraints. This enables type parameter inference from structural constraints.

### Solver Runs Per-Module
Each module's functions are solved independently. Debug traces from stdlib and test module are interleaved but separate.

## Worked Example

```kestrel
let x: Int? = 5
```

### Generation Phase
```
Register Optional[Int64] type (from annotation)
Register literal 5 type (Infer₁)
  → Conforms(Infer₁, ExpressibleByIntLiteral)
  → Promotable(Infer₁, Optional[Int64], expr_id)
```

### Solving

**Round 1:**
- `Conforms(Infer₁, ExpressibleByIntLiteral)` → Infer₁ not resolved → **Defer**
- `Promotable(Infer₁, Optional[Int64])` → Infer₁ is literal + target is potential promotion → **Defer**

**No progress. Apply literal defaults.**

**Default Round:**
- Infer₁ has `ExpressibleByIntLiteral` → apply default `Int64`
- Substitute Infer₁ → Int64

**Round 2:**
- `Conforms(Int64, ExpressibleByIntLiteral)` → oracle says yes → **Solved**
- `Promotable(Int64, Optional[Int64])` → can't unify directly → check `FromValue[Int64]` on `Optional` → found → record promotion → **Solved**

### Solution
```
types:      { Infer₁ → Int64 }
promotions: { expr_id → PromotionInfo(Optional[Int64], Optional.from, {T: Int64}) }
```

### Apply Phase
```
Before: Let(x, Literal(5))
After:  Let(x, Call(Optional.from, [Literal(5)]))
        with type: Optional[Int64]
```
