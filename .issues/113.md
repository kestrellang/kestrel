## Description

## Summary

Tracking issue for the MIR rewrite ("MIR-3") on the `refactor/mir` branch — a new OSSA-form MIR with a uniform ownership model, replacing the previous MIR.

## Scope

- **OSSA MIR** — ownership-SSA form where every value is `@owned` or `@guaranteed` (no `Ownership::None`); verifier enforces the invariants.
- **Ownership-based memory model** — drop elaboration, drop shims, and clone shims synthesized from types; per-instantiation copy semantics (Copyable/Cloneable) folded through the solver, MIR, and semantics.
- **Monomorphization** — `mono`/`expand` pipeline, layout dedup, drop/clone-shim expansion per instantiation.
- **Backends** — cranelift (default at `-O0`) and the LLVM backend (#78) both driven off this MIR.
- **Move checking** — `kestrel-analyze` move tracking restored on top of the new pipeline.

## State

Substantially complete and stabilizing — hence **Beta**. The full test suite is passing on `refactor/mir` (~3053/1), the LLVM backend validates identically to cranelift, and `kestrel-wall` builds and runs end-to-end (GET/POST/SQLite). Remaining work is residual memory-model edge cases and stabilization before merge.
