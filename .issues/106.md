# Issue #106: Mutating/inout closure parameters (in-place COW mutation)

**Author:** @jkpdino
**Created:** 2026-06-02T16:12:56Z
**URL:** https://github.com/kestrellang/kestrel/issues/106

## Description

## Summary

Add **`mutating` (by-reference) closure parameters** so a closure can mutate its argument in place — enabling clean in-place mutation of copy-on-write storage. Today every COW mutator must clone the whole backing buffer.

## Motivation

All COW collections (`Array`, `String`, `Deque`, `Dictionary`, `Set`) mutate via a read-modify-writeback dance:

```kestrel
var s = self.storage.read();           // deep-CLONES the whole ArrayStorage (new buffer)
s.ptr.offset(by: s.len).write(element);
s.len = s.len + 1;
self.storage.setValue(s)               // writes the clone back
```

`RcBox.getValue()` cannot return an owned non-`Copyable` value without cloning (OSSA forbids a bitwise alias — `lang.ptr_read` of a non-`Copyable` is "copy of non-Copyable value"). So **every mutation clones the entire buffer → O(n) per op → O(n²)** for n appends.

(A memory **leak** from this path — the orphaned pre-mutation buffer — was just fixed by a stopgap: `RcBox.setValue` now `dropInPlace()`s the old occupant before writing. That stops the leak but leaves the per-mutation clone.)

The clean fix is to mutate the heap value in place:

```kestrel
self.storage.modify { it.len += 1; it.ptr.offset(by: old).write(element) }   // zero clone
```

## What's blocked today (empirically verified)

1. Plain closure parameters are immutable `let` → `{ (s) in s.len += 1 }` ⇒ `E200 cannot assign to immutable variable`.
2. `mutating`/`inout` in a closure/function-**type** parameter position is a **parse error** (`expected ')', 'some', or 4 others, found 'mutating'`). `mutating` is accepted only on `func`/method params today; `inout` is not in the language.
3. Escaping a field pointer out of the read-only `with` borrow — `valuePtr().with { (s) in Pointer(to: s.len) }.write(x)` — compiles but **crashes** (SIGILL): `ptr_read` lowers to `Op::PtrRead`, a `@guaranteed`, non-escaping, immutable borrow.

## Work required

- **parser** — accept `mutating` (and/or introduce `inout`) on closure/function-type params and closure-literal params.
- **type-infer** — model mutable-borrow closure params; enforce exclusivity/uniqueness.
- **mir** — thread a `BeginMutBorrowAddr` (already exists in `kestrel-mir-3`) of the pointee **through** the closure call into the body, so `it.len = …` lowers to in-place `field_addr`+store, not clone+writeback.
- **stdlib** — add `Pointer.withMut` / `CowBox.modify` / `RcBox.modify`; rewrite the ~16 `array.ks` mutators (+ `String`/`Deque`/`Dictionary`/`Set`) from `read()`/`write()`+`setValue` to `modify`. The `setValue` `dropInPlace` stopgap can then be revisited.

**Alternative / companion:** a `ptr_take` / `TakeAddr` move-out primitive (destructive read leaving the slot uninit) so the read-out moves instead of clones.

## Verification

`MallocStackLogging=1 leaks --atExit -- <bin>` should stay at 0 leaks for mutation paths, with correctness asserted via read-back (e.g. `append` then `count`/subscript), plus guard-malloc for double-free.
