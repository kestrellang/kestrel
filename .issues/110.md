# Issue #110: Add `Terminable` protocol for `@main` return types

**Author:** @jkpdino
**Created:** 2026-06-04T02:50:15Z
**URL:** https://github.com/kestrellang/kestrel/issues/110

## Description

## Summary

Introduce a `Terminable` protocol so a `@main` function can return any type that knows how to produce a process exit code, generalizing the fixed `Void`/`Int64` return shipped in #109.

## Motivation

In 0.16, `@main` (#109) keeps the current entry ABI — it may return `Void` or `Int64`. That's limiting: we want `main` to be able to return richer types (and to throw), with each deciding its own exit code, the way Rust's `Termination` trait works.

## Design

Protocol with a single **computed property**:

```kestrel
protocol Terminable {
    var exitCode: Int32 { get }
}
```

`@main func main() -> T` requires `T: Terminable`. The compiler synthesizes the real C entry point as a wrapper:

```
int main(argc, argv):
    <top-level static init>     // moved out of inject_init_call_into_main
    return run().exitCode
```

This replaces the special-cased `Void`/`Int64` entry ABI from #109 (and the older `return_mode(is_main)` / `is_main_function` machinery) with one synthesized wrapper that reads `.exitCode` off whatever concrete type `@main` returns.

### Stdlib conformances

- `Int32` → `self`
- `Void` → `0` (decide: `extend Void: Terminable`, or a compiler assist if `Void` can't be extended)
- `Result[T: Terminable, E: Error]` → `.Ok(t)` → `t.exitCode`; `.Err(e)` → print `e` to stderr, return nonzero

### Throwing main

`@main func main() throws -> T` flows through the `Result` conformance: an uncaught error is printed to stderr and becomes a nonzero exit code. Error bound assumed `E: Error` (so it carries a printable message).

## Decisions locked

- `exitCode` is a computed property, not a method.
- Throwing `main` supported, routed through `Result[T: Terminable, E: Error]: Terminable`.

## Open questions

- Can `Void` conform to a protocol in Kestrel, or does the wrapper special-case a `Void` return → 0?
- Exact error bound for throwing `main` (`E: Error` vs. best-effort print of any type).

## Depends on

- #109 (`@main` attribute) — this issue generalizes its return type.
