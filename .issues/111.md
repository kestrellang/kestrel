# Issue #111: Add `@inline` attribute (LLVM inlining attrs)

**Author:** @jkpdino
**Created:** 2026-06-04T02:50:17Z
**URL:** https://github.com/kestrellang/kestrel/issues/111

## Description

## Summary

Wire up an `@inline` attribute that maps to LLVM inlining attributes, with an enum-style argument: `@inline(.Always)`, bare `@inline`, and `@inline(.Never)`.

## Motivation

Profiling `kestrel-wall` showed roughly 72% of uncached request time going to un-inlined scalar/pointer primitive calls — inlining-bound work. An explicit `@inline(.Always)` on hot primitives lets us force those to inline.

## Design

Argument mapping to LLVM:

| Attribute | LLVM attribute |
|---|---|
| `@inline(.Always)` | `alwaysinline` |
| `@inline` (bare) | `inlinehint` |
| `@inline(.Never)` | `noinline` |

### The enum is conceptual

Attributes are processed at the AST stage, before type inference, so `.Always` / `.Never` can't resolve to a real stdlib enum member there. The attribute validator matches the case **names** (`Always`/`Never`) by string and rejects anything else with a diagnostic. We can document an `InlineMode { Always, Never }` for the LSP/docs, but the compiler matches by name — there is no real enum to resolve.

### Already parses

`@inline` and `@inline(...)` already lex/parse into `Attribute { name: "inline", args }` (`kestrel-ast-builder/src/build.rs`). What's missing is a consumer that lowers the attribute to LLVM function attributes in the codegen backend.

## Backend story

0.17 maps optimization level to backend:

- `-O0` → cranelift (fast compile; **does not inline**, so `@inline` is a no-op here)
- `-O1` / `-O2` → LLVM (honors `@inline`)

So `@inline` is only live on optimized builds — which is where it matters (and what the wall ships). This is the change that makes the attribute real rather than dead plumbing.

## Decisions locked

- LLVM attributes only (no MIR-level inliner pass).
- Enum-style argument `.Always` / bare / `.Never`, matched by name.
- `-O0` cranelift / `-O1`,`-O2` LLVM backend mapping.

## Diagnostics

- `@inline` on a non-callable declaration.
- `@inline(...)` with an argument other than `.Always` / `.Never`.

## Scope

- Lower `@inline` to LLVM `alwaysinline` / `inlinehint` / `noinline` in the LLVM codegen backend.
- Implement the opt-level → backend mapping (`-O0` cranelift, `-O1`/`-O2` LLVM).
- Attribute validation + diagnostics.
