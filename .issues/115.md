## Description

## Summary

Add **second-class references** to Kestrel: `&T` (shared/immutable borrow) and `&mutating T` (mutable borrow), reusing the existing `@owned`/`@guaranteed`/`@borrow` convention machinery rather than a new mechanism. "Second-class" = scoped, not freely storable in arbitrary long-lived locations. `&mutating` is **allowed to alias** (no Rust-style exclusivity / two-phase borrow / non-aliasing inference).

## Motivation

Pass-by-reference without COW clones, accessor methods returning borrows, and the `Str.toBytes()`-class amplifiers — most of the day-to-day value comes from Stage 1 alone.

## Staging

- **Stage 1** — references within a function; return references; lifetimes inferred; the referenced value must escape the function to be returnable.
- **Stage 2** — every type carries a lifetime; references captured by closures, placed inside tuples/structs/enums. **Cost center / make-or-break.**
- **Stage 3** — references on protocols; default `Static` bound on every type param; `T: not Static` allows referential lifetimes.

## Difficulty (1 expert, ~8–12 months end-to-end)

| Stage | Difficulty | Effort | Biggest risk |
|---|---|---|---|
| 1 — in-fn refs, return refs, inferred lifetimes | Very-high | ~10–14 wk, ~20–35 files | New escape/outlives checker is the only thing preventing **silent dangling returns** |
| 2 — lifetime on every type; refs in aggregates/closures | Very-high (cost center) | ~16–26 wk, ~30–40 files | `MonoTypeKey` lifetime-provenance collapse; aliased `&mutating` UAF; collision with Rc-closure migration |
| 3 — refs on protocols, default `Static`, `T: not Static` | High (easiest) | ~4–7 wk, ~12–16 files | False-positive structural is-`Static`; witness-lowering lifetime-erasure mismatch |

**Key insight:** the MIR substrate (borrow instruction set, `ParamConvention::Borrow/MutBorrow` → `PassMode::ByRef`, `ValueDef.borrow_source`, block-param forwarding of `@guaranteed`) is far more reusable than the staging implies. What's missing is **analysis the compiler deliberately doesn't have** — the OSSA verifier never proves a `@guaranteed` value's source is live in successor blocks, which is why returns unconditionally copy `@guaranteed → @owned` today. The real net-new cost is a **region/outlives + escape checker**, not IR plumbing.

## Minimal-viable Stage 1 (recommended first deliverable)

Ship reference *parameters* (nearly free) + in-function intra-block borrows (already work) + **return-borrow-of-a-parameter**, shared `&T` only. A 6-site change gated on one `ret_borrow: bool`. Defer storable `MirTy::Ref`, region/outlives solver, borrows-of-locals returns, cross-block forwarding, `&mutating` returns, ref-returning closures/protocol methods.

Two non-negotiable commitments made in Stage 1: (1) a distinct `MirTy::Ref` — **never reuse `MirTy::Pointer`** (it carries no provenance, and squatting turns Stage 2 into a migration); (2) build the escape/outlives checker as the *first* deliverable — without it everything compiles clean and dangles at runtime.
