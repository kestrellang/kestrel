# Issue #109: Add `@main` attribute for explicit entry points

**Author:** @jkpdino
**Created:** 2026-06-04T02:49:20Z
**URL:** https://github.com/kestrellang/kestrel/issues/109

## Description

## Summary

Add a `@main` attribute that explicitly marks the program entry point, replacing today's discovery-by-name (a function literally called `main`).

## Motivation

Right now the entry point is found by name — the compiler treats any function named `main` (or `*.main`) as the C entry (`kestrel-mir-lower/src/items/static_lower.rs:85`, `kestrel-codegen-cranelift/src/context.rs:110`). This is implicit, can't live under a different name, and gives flock no clean hook to select an entry per binary target. An explicit `@main` makes the entry point a deliberate, discoverable marker.

## Design (0.16)

- `@main` goes on a **free function**, e.g. `@main func main() { ... }`.
- **Explicit only** — name-based discovery is removed. A function named `main` without `@main` is just an ordinary function.
- **Return type stays as it is today**: `Void` or `Int64` (the current main ABI / exit-code behavior is unchanged). Generalizing the return type to an arbitrary type via a protocol is deferred to 0.17 (see Terminable issue, linked below).
- The "an executable build needs exactly one `@main`" rule is scoped to **executable builds only** — libraries (and type-checking-only runs like diagnostics tests) need no `@main`.

## Decisions locked

- Free function, not a type (no Swift-style `@main struct`).
- Explicit only; no implicit `func main` fallback.
- 0.16 keeps the current `Void`/`Int64` return ABI.

## Diagnostics

- No `@main` in an executable build.
- More than one `@main` in a build.
- `@main` on a non-function, or (0.16) a `@main` whose return type isn't `Void`/`Int64`.

## Migration cost

Requiring explicit `@main` breaks every `.ks` that relies on `func main` as the entry:

- **583 execution tests** in `kestrel-test-suite/testdata` (these build + run) — each needs `@main` added. Mechanical sweep (`func main` → `@main\nfunc main`); scriptable.
- **32** files in `examples/`, `lang/*` bins (incl. flock itself), `external/`.
- **224 diagnostics tests** use `func main` but only type-check — protected by scoping the entry-point requirement to executable builds, so they should not need changes (verify no new diagnostic fires at them).

## Implementation pointers

- `@inline`/attribute parsing already exists: ast-builder produces `Attribute { name, args }` (`kestrel-ast-builder/src/build.rs`). `@main` is a new recognized attribute name.
- Replace name-based discovery in `static_lower.rs` (entry selection + `inject_init_call_into_main`) and cranelift `is_main_function` / `return_mode(is_main)` (`abi.rs`).

## Out of scope (follow-ups)

- Generalized return type via the `Terminable` protocol + synthesized `.exitCode` wrapper + throwing `main` → **0.17**.
- `@main` parameters (argv/env) — TBD; 0.16 is parameterless.
