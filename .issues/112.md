# Issue #112: Binary targets in flock (`[[bin]]`, Cargo-style)

**Author:** @jkpdino
**Created:** 2026-06-04T02:50:18Z
**URL:** https://github.com/kestrellang/kestrel/issues/112

## Description

## Summary

Teach flock to build binary targets, Cargo-style: a default binary from `src/main.ks`, extra binaries from `src/bin/*.ks`, and explicit `[[bin]]` entries in `flock.toml`.

## Motivation

Today the package model (`flock.toml`) is consumed only by the LSP and the registry; the Rust `kestrel build` just takes raw `.ks` files on the command line, with no concept of targets, dependency resolution at build time, or multiple executables. flock (the package manager, written in Kestrel under `lang/flock/`) is where binary targets belong.

## Design (Cargo-style convention)

- `src/main.ks` → default binary, named after the package.
- `src/bin/*.ks` → one binary each.
- `[[bin]] { name, path }` in `flock.toml` overrides / adds targets.

### Compilation model

Whole `src/` compiles per bin: each binary build compiles all of `src/` together (the package library + that one entry file), excluding the other bins' entry files. No separate lib/bin module boundary — simpler for flock, at the cost of recompiling shared code per bin.

### Orchestration

flock stays the orchestrator: it invokes `kestrel build` once per binary with `{ that bin's entry } + { the package's shared sources }` and `-o <binname>`. The compiler change is minimal — it just needs `@main` (#109) to be the per-build entry selector so a multi-file build has exactly one entry.

## Decisions locked

- Cargo-style convention (`src/main.ks`, `src/bin/*.ks`, `[[bin]]` override).
- Whole `src/` compiles per bin (no lib/bin module boundary).

## Open questions

- `[[bin]]` manifest schema and whether a package can be lib-only / bin-only / both.
- How `flock.toml` parsing (`lang/flock/src/manifest.ks`) and discovery (`discover.ks`) surface targets to the build graph (`graph.ks`).

## Depends on

- #109 (`@main` attribute) — each binary needs exactly one `@main` as its entry selector.
